## 二分答案

### 1.第N个神奇数字 https://leetcode.cn/problems/nth-magical-number/

**思路：主要是怎么想到二分答案，以及check函数怎么写。二分主要是本题具有单调性，然后check函数就是要用容斥原理来写，然后gcd和lcm可以用python内置的，但如果比赛的python比较老，就得自己手搓了**

**代码：**
```python
class Solution:
    def nthMagicalNumber(self, n: int, a: int, b: int) -> int:
        mod = 10 ** 9 + 7 

        def gcd(a, b):
            return a if b == 0 else gcd(b, a % b)
        
        def lcm(a, b):
            return a // gcd(a, b) * b 

        def check(mid):
            return mid // a + mid // b - mid // lcm(a, b) >= n

        l, r = 2, n * min(a, b)
        while l <= r:
            mid = (l + r) >> 1 
            if check(mid):
                r = mid - 1
            else:
                l = mid + 1 
        
        return l % mod 
```

### 2.分割数组的最大值(画匠问题) https://leetcode.cn/problems/split-array-largest-sum/description/

**思路：就是最小化最大值问题，用二分解决。不过需要注意，我们的check函数是依据二分来写的，而不是二分依赖于check函数来写，这会直接决定答案的正确性**

**代码：**
```python
class Solution:
    def splitArray(self, nums: List[int], k: int) -> int:
        
        def check(mid):
            s, cnt = 0, 1
            for x in nums:
                if s + x <= mid:
                    s += x
                else:
                    cnt += 1
                    s = x
            return cnt <= k

        l, r = max(nums), sum(nums)
        while l <= r:
            mid = (l + r) >> 1
            if check(mid):
                r = mid - 1
            else:
                l = mid + 1
        return l
```

### 4.找出第K小的数对距离 https://leetcode.cn/problems/find-k-th-smallest-pair-distance/description/

**思路：首先每个数的位置并不重要，重要的是数之间的差值。因此我们就可以先排个序，然后二分答案。此时的check函数中就写两数差值>mid的不定长滑窗，类似于滑窗，因为他只关注两个端点值。算出其个数之后，就与k进行比较，这个很像画匠问题的处理**

**代码：**
```python
class Solution:
    def smallestDistancePair(self, nums: List[int], k: int) -> int:
        nums.sort()

        def check(mid):
            l = cnt = 0 
            for r, x in enumerate(nums):
                while x - nums[l] > mid:
                    l += 1
                cnt += r - l 
            return cnt >= k

        l, r = 0, 10 ** 6 + 1
        while l <= r:
            mid = (l + r) >> 1
            if check(mid):
                r = mid - 1
            else:
                l = mid + 1
        return l
```

### 5.同时运行n台电脑的最长时间 https://leetcode.cn/problems/maximum-running-time-of-n-computers/description/

**思路：也是二分答案最长的运行时间，因为他有单调性，那么check函数该怎么写？注意到sum(碎片化电池) >= mid * 无法单独供电的电脑数时，能够实现mid运行时间的供电，那么就在满足上述条件时，向更大的mid进行二分即可**

**为什么左神代码中的check函数，对于大电池来说，他的剩余电量不能再利用？因为一个电池只能给一台电脑供电，当前电脑独占了大电池，所以说剩余的电量无法再被利用**

**代码：**
```python
class Solution:
    def maxRunTime(self, n: int, batteries: List[int]) -> int:

        def check(mid):
            return sum(min(mid, b) for b in batteries) >= n * mid

        l, r = 0, sum(batteries) 
        while l <= r:
            mid = (l + r) >> 1
            if check(mid):
                l = mid + 1
            else:
                r = mid - 1
        return r
```

### 6.移山所需要的最小秒数 https://leetcode.cn/problems/minimum-number-of-seconds-to-make-mountain-height-zero/description/?envType=daily-question&envId=2026-03-13

**思路：二分答案 + 数学推导。答案的秒数具有单调性，越大越能满足条件，反之越不能满足条件，然后就是等差数列求和公式以及二次函数推导出秒数可以移除的山高度，之后就可以进行二分答案了**

**代码：**
```python
class Solution:
    def minNumberOfSeconds(self, mountainHeight: int, workerTimes: List[int]) -> int:
        def check(mid):
            cnt = 0 
            for x in workerTimes:
                tmp = 2 * mid // x 
                cnt += (-1 + isqrt(1 + 4 * tmp)) // 2
                if cnt >= mountainHeight:
                    return True 
            return False

        l, r = 1, 10 ** 18
        while l <= r:
            mid = (l + r) >> 1
            if check(mid):
                r = mid - 1 
            else:
                l = mid + 1
        return l
```

### 7.完成所有任务的最少初始能量 https://leetcode.cn/problems/minimum-initial-energy-to-finish-tasks/description/?envType=daily-question&envId=2026-05-12

**思路：排序贪心 + 二分答案/公式推导。首先观察题目，可以发现这个能量具有单调性，初始能量越多，那么完成的任务越多；反之越少；然后就是check函数怎么写了，我们希望task[0]尽可能小，而task[1]尽可能大，所以说两者相减得到整体越小越好，因此就按照这个进行排序，最后跑一个二分答案即可**

**代码：**
```python
class Solution:
    def minimumEffort(self, tasks: List[List[int]]) -> int:
        n = len(tasks)
        tasks.sort(key=lambda x: x[0] - x[1])

        def check(mid):
            for x, y in tasks:
                if mid < y:
                    return False 
                mid -= x 
            return True

        l, r = 1, 10 ** 18 
        while l <= r:
            mid = (l + r) >> 1
            if check(mid):
                r = mid - 1
            else:
                l = mid + 1
        return l 
```

## 浮点数二分

### 1.圆覆盖 https://www.nowcoder.com/practice/4f96afe5dfe74dad88dbe419d33f9536?channelPut=tracker2

**思路：半径具有单调性，半径越大，覆盖的面积越大，越容易满足条件；否则半径越小，越不容易满足条件。那就可以用二分，对于本题有要求的精度来说，就需要用到浮点数二分**

'''

def solve():

    a = []
    total = 0
    mx = 0.0
    for _ in range(n):

    solve()

## 三分找极值点

### 1.qcjj寄快递 https://www.nowcoder.com/practice/229bbec64b6b42e48171bef8f88ada47?channelPut=tracker2

**思路：浮点数三分。题目给定了单峰函数，并且每个单峰函数的$k_i$相互独立。所以说我们每两个点之间算出一个单峰函数，之后再用浮点数三分把$k_i$给算出来，并且算出具体的函数值$t_i$，接着不断累加到ans中，最终输出答案即可**

**技巧：三分函数我们只需要写找最大值单峰的逻辑即可，然后用一个find_max变量来记录我们是否找的是最大值，如果是的话就为True，直接用我们写的找最大值单峰逻辑即可；否则为False，就意味着我们要三分的函数为最小值单峰函数，那就可以整体添加一个-号，把他变成最大值单峰函数，之后就可以复用找最大值单峰的逻辑了。最终，找到并输出最大值/最小值对应的自变量，以及最大值/最小值即可**

**代码：**
```python
import sys
input = lambda: sys.stdin.readline().strip()

import math
inf = 10 ** 18

def I():
    return input()

def II():
    return int(input())

def MII():
    return map(int, input().split())

def LI():
    return input().split()

def LII():
    return list(map(int, input().split()))

def LFI():
    return list(map(float, input().split()))

fmax = lambda x, y: x if x > y else y
fmin = lambda x, y: x if x < y else y
isqrt = lambda x: int(math.sqrt(x))

'''

'''

def ternary_search(func, l, r, eps=1e-9, find_max=True):
    f = func if find_max else lambda x: -func(x)

    while r - l > eps:
        m1 = l + (r - l) / 3
        m2 = r - (r - l) / 3

        f1, f2 = f(m1), f(m2)

        if f1 < f2:
            l = m1
        else:
            r = m2

    mid = (l + r) / 2
    return mid, func(mid)

def solve():
    n = II()

    def dis(x1, y1, x2, y2):
        return math.sqrt(abs(x2 - x1) * abs(x2 - x1) + abs(y2 - y1) * abs(y2 - y1))

    ans = 0
    pre_x = pre_y = -1
    for i in range(n):
        x, y = LFI()
        if i == 0:
            pre_x, pre_y = x, y
            continue

        e = dis(pre_x, pre_y, x, y)
        func = lambda k: 2 * k + 2 * e / (2 ** k)
        ans += ternary_search(func, 0, 1e3, find_max=False)[1]
        pre_x, pre_y = x, y

    print(ans)

t = 1
# t = II()
for _ in range(t):
    solve()
```

## 二分找第K小的数

### 1.乘法表中第K小的数 https://leetcode.cn/problems/kth-smallest-number-in-multiplication-table/description/

**思路：和一般的TopK问题不同，本类问题中的数具有单调性，因此可以用二分的思想。具体来说就是：第K小的数就是要找的这个数，前面有K个数比他小，当要找的这个数x越大，那么它前面的数越多，这就具有单调性，然后写二分即可**

**代码：**
```python
class Solution:
    def findKthNumber(self, m: int, n: int, k: int) -> int:
        
        def check(mid):
            cnt = 0 
            for i in range(1, m + 1):
                cnt += min(n, mid // i)
            return cnt >= k

        l, r = 1, m * n 
        while l <= r:
            mid = (l + r) >> 1
            if check(mid):
                r = mid - 1
            else:
                l = mid + 1
        return l
```

### 2.第k个缺失的正整数 https://leetcode.cn/problems/kth-missing-positive-number/

**思路：首先题目给定的是具有单调性的数组，并且k越大那么结果也越大，也具有单调性，因此可以用二分。那么就转化成了找第k小问题了，缺失的数目为$arr[mid] - 1 - mid$，也就是从$1$到$arr[mid] - 1$这些数中，还存在有$arr[0]$到$arr[mid - 1]$这$mid$个数，剩下的就是缺失的数**

**那么最终缺失的具体数字就是$l + k$，这是根据公式推导出来的，简单来说就是$a[l] - 1 - (a[l] - 1 - l - k) = l + k$，表示从当前这个缺失数$>= k$的l位置，往前倒退$a[l] - 1 - l - k$个单位，就是最终缺失的数了，可以代例子看看**

**代码：**
```python
class Solution:
    def findKthPositive(self, arr: List[int], k: int) -> int:
        l, r = 0, len(arr) - 1
        while l <= r:
            mid = (l + r) >> 1
            if arr[mid] - 1 - mid >= k:
                r = mid - 1 
            else:
                l = mid + 1
        return l + k
```

## 二分查找

### 1.寻找旋转排序数组中的最小值 https://leetcode.cn/problems/find-minimum-in-rotated-sorted-array/?envType=daily-question&envId=2026-05-15

**思路：二分查找。首先数组具有单调性，就得想到二分；然后这个单调数组进行了单调旋转，那么它依旧具有单调性，只需要每次mid和最后一个元素比较大小，就可以确定最小值在哪一边，然后不断二分得到答案**

**代码：**
```python
class Solution:
    def findMin(self, nums: List[int]) -> int:
        n = len(nums)
        l, r = 0, n - 1
        while l <= r:
            mid = (l + r) >> 1
            if nums[mid] > nums[-1]:
                l = mid + 1
            else:
                r = mid - 1
        return nums[r + 1]
```
