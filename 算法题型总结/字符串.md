### 1.月月查华华手机 https://www.nowcoder.com/practice/c2877187b1bb4573927e759a60f5d3e1?channelPut=tracker2

### 2.小红的数位删除 https://www.nowcoder.com/practice/a1044c31930341d6a51c9a619254074a?channelPut=tracker2

**思路：暴力枚举。注意到题目的a，b数字最长为10，因此可以枚举a和b的所有子序列，然后判断是否满足条件，时间复杂度为$O(2^{10} * 2^{10})$，为$1024 * 1024$大概是$10^{6}$量级左右，可以通过**

**具体来说，我们可以枚举保留的位数个数i和j，然后用组合数函数combinations(a, x)表示从a这个迭代器或者数组等这种存放元素的结构中，选出x个数进行组合。接着，我们就可以生成保留相应位数的数字na和nb了，然后判断一下是否满足“na % nb == 0 or nb % na == 0”的条件，就可以收集答案了（注意是收集删除次数）。最终，枚举完成后输出答案即可**

**注意：本题至少需要保留一个数位，不能变成空串，因为空串不能算成0**

**代码：**
```python

import math

def I():
    return input()

def II():
    return int(input())

    return list(map(float, input().split()))

isqrt = lambda x: int(math.sqrt(x))

'''

def solve():
    a, b = MII()
    sa, sb = str(a), str(b)
    m, n = len(sa), len(sb)

    ans = inf

    for i in range(1, m + 1):
        for pos_a in combinations(range(m), i):
            na = int(''.join(sa[p] for p in pos_a))

            for j in range(1, n + 1):
                for pos_b in combinations(range(n), j):
                    nb = int(''.join(sb[p] for p in pos_b))

                    if na % nb == 0 or nb % na == 0:
                        res = (m - i) + (n - j)
                        ans = fmin(ans, res)

    print(ans if ans < inf else -1)
```

### 3.不同的子序列II https://leetcode.cn/problems/distinct-subsequences-ii/

**思路：直接定义状态，然后分析状态转移方程。本题定义$cnt[i]$表示以处理了索引之后的i为结尾的字符，有多少个不同子序列。然后我们还需要两个变量：1.ans变量表示总的不同子序列个数；2.new_add变量表示每次往后扩展时，新增的不同子序列个数。最终，统计完所有不同子序列的数量之后，再-1也就是减去空集，然后取mod就是答案了**

**代码：**
```python
class Solution:
    def distinctSubseqII(self, s: str) -> int:
        mod = 10 ** 9 + 7 
        cnt = [0] * 26 
        ans = 1
        for x in s:
            new_add = (ans - cnt[ord(x) - ord('a')] + mod) % mod 
            ans = (ans + new_add) % mod 
            cnt[ord(x) - ord('a')] = (cnt[ord(x) - ord('a')] + new_add) % mod 
        return (ans - 1 + mod) % mod
```

### 2.小红的好01串修改 https://ac.nowcoder.com/acm/problem/297760

**思路：继续沿用这类题型的思路。我们执行以步长为1，次数为距离的操作，就可以翻转对应距离的一对元素。那么就可以分别计算反转成"0101..."和"1010..."这两种情况，所对应的操作次数，最终取min即可**


### 3.小美的01串翻转 https://www.nowcoder.com/practice/a0effc0e1dd24412a8f47c5dde0da075?channelPut=tracker2

**思路：外面两层for的暴力枚举字串，这个没法优化，需要优化的是内部的for算最小代价，这个比较巧妙。具体来说就是：从长度进行优化，令$c0 = 0$; $c1 = 0$表示两种不同的交替形式，然后就可以根据长度$j - i$的奇偶性加上异化，来判断当前位置匹配格式所需要花费的代价，ans每次累加更小的字串代价，就可以得到最终ans了**

**代码：**
```python
import sys
input = lambda: sys.stdin.readline().strip()

import math
inf = 10 ** 18

def I():
    return input()

def II():
    return int(input())

def MII():
    return map(int, input().split())

def LI():
    return input().split()

def LII():
    return list(map(int, input().split()))

def LFI():
    return list(map(float, input().split()))

fmax = lambda x, y: x if x > y else y
fmin = lambda x, y: x if x < y else y
isqrt = lambda x: int(math.sqrt(x))

'''

'''

def solve():
    s = I()
    n = len(s)
    a = list(map(int, s))

    ans = 0
    for i in range(n):
        c0 = c1 = 0
        for j in range(i, n):
            if (j - i) & 1:
                c0 += a[j] ^ 1
                c1 += a[j] ^ 0
            else:
                c0 += a[j] ^ 0
                c1 += a[j] ^ 1
            ans += fmin(c0, c1)
    print(ans)

t = 1
# t = II()
for _ in range(t):
    solve()
```

## 平衡01串问题

**通用思路：正难则反，奇偶性和异或的关系**

### 1.Balanced 01-String https://www.nowcoder.com/practice/cbd14d9770ca4efb98ad690272d76757?tpId=391&contestId=126798&channelPut=tracker1

**思路：很好的01字符串思维题，正难则反。题目说的是相邻相同的字符个数要为偶数，那么我们可以先算相邻不同字符个数的奇偶性，然后用(n - 1 - 相邻不同奇偶性) % 2来进行计算，那么相邻不同奇偶性怎么计算？**

**首先我们可以算相邻数异或的结果，然后把他们累加起来，再%2算其奇偶性，但由于异或与奇偶性的性质，上面的式子其实就是把所有式子都给异或起来，两者是恒等关系，比如：（s1 ^ s2 + s2 ^ s3 + s3 ^ s4）% 2 = s1 ^ s2 ^ s2 ^ s3 ^ s3 ^ s4**

**那么我们就可以得到结论：相邻不同的奇偶性由头尾异或和决定，相邻相同的奇偶性由n - 1和头尾异或和决定，那么方案数就好算了，直接判断下中间的'?'数量，然后判断下头尾的'?'情况即可**

**代码：**

## 交替01串问题

### 1.使二进制字符串字符交替的最少反转次数 https://leetcode.cn/problems/minimum-number-of-flips-to-make-the-binary-string-alternating/description/?envType=daily-question&envId=2026-03-07

**思路：主要是怎么想到转化成定长滑动窗口的，具体如下：首先操作1没有限制次数，那就可以操作任意次数，其操作出来的结果其实是s + s中长为n的滑动窗口字串；然后就是交替的处理方式，对于"010101..."来说其索引奇偶性和对应的数值有相等关系；反之对于"101010..."来说其索引奇偶性和对应的数值有不相等关系。因此，我们就做一个长度为n的定长滑动窗口，之后统计其中不相等/相等的位置数量，他就是要操作的次数，之后再用长度n去减，就得到相等/不相等的位置数，这两个取min就是本次的最小操作次数，然后再和ans不断取min就可以得到最终答案了**

**代码：**
```python
class Solution:
    def minFlips(self, s: str) -> int:
        inf = 10 ** 18 
        ans = inf
        n = len(s)
        cnt = 0 
        for i in range(2 * n - 1):
            if int(s[i % n]) != i & 1:
                cnt += 1
            l = i - n + 1
            if l < 0:
                continue 
            ans = min(ans, cnt, n - cnt)
            if int(s[l]) != l & 1:
                cnt -= 1
        return ans 
```

### 2.特殊的二进制字符串 https://leetcode.cn/problems/special-binary-string/description/?envType=daily-question&envId=2026-02-20

**思路：题目说了给定的特殊二进制字符串具有如下性质：1.二进制字符串中所有的'0'和'1'数量相同；2.前缀'1'的数量总是大于前缀'0'的数量。综合这两个性质，他其实和合法括号是一样的，合法括号也是这两个性质。因此，我们可以把'0'当成右括号，把'1'当成左括号，这样就实现了两者的转化**

**那具体该如何得到最大字典序字符呢？我们有两种情况可以进行讨论：1.拼接，我们把所有合法的小括号序列进行从大到小排序，之后在拼接起来，就可以得到最大的字典序；2.嵌套，对于一般的合法小括号序列来说，嵌套的字典序会比不嵌套的更大，所以说我们在遇到合法小括号的时候，就可以进行嵌套处理，即外面再套一层。由于我们可以进行任意次操作，所以说这个是可以实现的；并且套了一层之后，其内部剩余的字符串，我们也是采用相同的处理方法，那说明就改用递归去解决**

**最终，我们收集完所有合法并且最大化的小括号序列后，再用前面提到的排序拼接，就可以得到最终答案了**

**代码：**
```python
class Solution:
    def makeLargestSpecial(self, s: str) -> str:
        if len(s) <= 2:
            return s 
        
        ans = []
        diff = 0 
        start = 0 
        for i, c in enumerate(s):
            if c == '1':
                diff += 1
            else:
                diff -= 1
                if diff == 0:
                    ans.append('1' + self.makeLargestSpecial(s[start + 1:i]) + '0')
                    start = i + 1
                    
        ans.sort(reverse=True)
        return ''.join(ans)
```

### 3.最长有效括号 https://leetcode.cn/problems/longest-valid-parentheses/description/

**思路：没有记忆化，直接定义状态并且分析出状态转移方程，本题是定义$dp[i]$表示以$s[i]$字符为结尾的最长有效括号**

**代码：**
```python
class Solution:
    def longestValidParentheses(self, s: str) -> int:
        if not s:
            return 0

        n = len(s)
        s = list(s)
        dp = [0] * n 
        ans = 0
        for i, x in enumerate(s):
            if x == ')':
                pre = i - 1 - dp[i - 1]
                if pre >= 0 and s[pre] == '(':
                    dp[i] = dp[i - 1] + 2 + (dp[pre - 1] if pre > 0 else 0)
        return max(dp)
```

## KMP算法

### 1.找出字符串中第一个匹配项的下标 https://leetcode.cn/problems/find-the-index-of-the-first-occurrence-in-a-string/description/

**思路：非常标准的KMP算法板子题，其时间复杂度为$O(m + n)$，不过用自带的find方法会更快一点。对于KMP算法来说，核心点有两个：1.next数组和含义及快速跳转生成；2.next数组是如何加速字符串的匹配的（KMP算法流程）**

**代码：**
```python
class Solution:
    def strStr(self, s: str, t: str) -> int:
        m, n = len(s), len(t)
        
        def kmp(s, t):
            nxt = [0] * n
            nxt[0] = -1
            
            i = 2
            cn = 0 
            while i < n:
                if t[i - 1] == t[cn]:
                    nxt[i] = cn + 1
                    cn += 1
                    i += 1
                elif cn > 0:
```

**思路：AC自动机板子题。简单来说，他其实是在前缀树的基础上利用KMP的思想，引入了fail指针，实现多词频的统计。本题的模板已经是经过优化后的代码，可以直接使用，减少了fail指针的绕圈**

## Manacher算法

### 1.Manacher https://www.luogu.com.cn/problem/P3805

**思路：经典Manacher算法板子题，简单讲一下这个算法：首先要用'#'符号进行填充，这样能够处理掉奇偶长度回文带来的额外判断。然后就是三个重要参数：1.p数组表示以i位置为中心，其回文半径的长度；2.r表示能够扩展的最右回文边界；3.c表示产生最早的最右扩展索引**

**接着就是4种情况：1.如果当前i >= r，那么直接进行扩展；2.如果i < r，并且对称点的左边界在r内部，那么p[i] = p[2 * c - i]；如果i < r，并且对称点的左边界超出c的回文范围，那么p[i] = r - i；如果i < r，并且对称点的左边界刚好在c的回文范围，那么也要进行扩展。最终，最长回文子串的长度就是最大回文半径 - 1**

**代码：**

### 2.最长回文子串 https://leetcode.cn/problems/longest-palindromic-substring/

**思路：Manacher算法。需要知道两个结论：1.回文半径长度 - 1就是最大的回文子串长度；2.扩展后的字符串结尾 // 2，就是原字符串的结尾。因此，求两个参数mx，end就可以知道结尾位置及长度，然后就可以从原字符串中切片出答案了**

**代码：**
```python
class Solution:
    def longestPalindrome(self, s: str) -> str:
        ss = '#' + '#'.join(list(s)) + '#'
        n = len(ss)

        p = [1] * n 
        c = r = 0 
        mx = end = 0
        for i in range(n):
            l = min(r - i, p[2 * c - i]) if r > i else 1 
            while i + l < n and i - l >= 0 and ss[i - l] == ss[i + l]:
                l += 1
            if i + l > r:
                r = i + l 
                c = i 
            if l > mx:
                mx = l - 1
                end = r // 2
            p[i] = l 
        
        return s[end - mx:end]
```

### 4.不重叠回文子字符串的最大数目 https://leetcode.cn/problems/maximum-number-of-non-overlapping-palindrome-substrings/

**思路：Manacher算法 + 贪心。首先整体字符串进行ss改写，之后就遍历一遍，一旦发现有满足l > k条件的字符串就进行断开。断开之后就ans + 1说明发现一个答案，然后在新的索引上，向后继续进行Manacher算法，最终返回答案即可。不过要注意一点，新的索引一定要是'#'，否则后续会造成交叉问题，导致答案统计出错**

**代码：**
```python
class Solution:
    def maxPalindromes(self, s: str, k: int) -> int:
        ss = '#' + '#'.join(list(s)) + '#'
        n = len(ss)
        p = [1] * n 
        
        def find(q):
            c = r = q
            for i in range(q, n):
                l = min(r - i, p[2 * c - i]) if r > i else 1 
                while i + l < n and i - l >= q and ss[i - l] == ss[i + l]:
                    l += 1
                    if l > k:
                        return i + k + (1 if ss[i + k] != '#' else 0)
                if i + l > r:
                    r = i + l 
                    c = i 
                p[i] = l 
            return -1 
            
        ans = 0 
        nxt = 0 
        while (nxt := find(nxt)) != -1:
            ans += 1
        return ans 
```

## 扩展KMP算法（z数组与e数组）

### 1.扩展KMP https://www.luogu.com.cn/problem/P5410

**思路：虽说是扩展KMP，但是算法过程和Manacher非常相似。定义z数组：$z[i]$表示字符串$s[i:]$与$s[0:]$匹配的长度；定义e数组：$e[i]$表示字符串$a[i]$与字符串$b[0:]$匹配的长度。算z数组有点类似于Manacher里面算p数组的过程，然后算e数组需要用到z数组，具体计算过程又类似于算z数组**

**代码：**
```python
import sys
input = lambda: sys.stdin.readline().strip()

import math
inf = 10 ** 18

def I():
    return input()

def II():
    return int(input())

def MII():
    return map(int, input().split())

def LI():
    return input().split()

def LII():
    return list(map(int, input().split()))

def LFI():
    return list(map(float, input().split()))

fmax = lambda x, y: x if x > y else y
fmin = lambda x, y: x if x < y else y
isqrt = lambda x: int(math.sqrt(x))

'''

'''

def solve():
    a = I()
    b = I()
    m, n = len(a), len(b)

    def z_arr(s):
        z = [0] * n
        z[0] = n

        c = r = 1
        for i in range(1, n):
            l = fmin(r - i, z[i - c]) if r > i else 0
            while i + l < n and s[i + l] == s[l]:
                l += 1
            if i + l > r:
                r = i + l
                c = i
            z[i] = l

        return z

    z = z_arr(b)
    ans_z = 0
    for i, x in enumerate(z, 1):
        ans_z ^= i * (x + 1)
    print(ans_z)

    def e_arr(a, b):
        e = [0] * m

        c = r = 0
        for i in range(m):
            l = fmin(r - i, z[i - c]) if r > i else 0
            while i + l < m and l < n and a[i + l] == b[l]:
                l += 1
            if i + l > r:
                r = i + l
                c = i
            e[i] = l

        return e

    e = e_arr(a, b)

    ans_e = 0
    for i, x in enumerate(e, 1):
        ans_e ^= i * (x + 1)
    print(ans_e)

t = 1
# t = II()
for _ in range(t):
    solve()
```

### 2.将单词恢复初始状态所需的最短时间 https://leetcode.cn/problems/minimum-time-to-revert-word-to-initial-state-ii/description/

**思路：z数组。利用z数组，得到字符串$s[i:]$与$s[0:]$匹配的长度，然后每k个长度进行一次跳跃，判断能否通过当前位置得到原有字符。最终，如果匹配失败，也就是越界了，那么操作n / k的上限次，必然能够恢复出原来的字符串**

**代码：**
```python
class Solution:
    def minimumTimeToInitialState(self, s: str, k: int) -> int:
        n = len(s)

        def z_arr(s):
            z = [0] * n 
            z[0] = n 

            c = r = 1 
            for i in range(1, n):
                l = min(r - i, z[i - c]) if r > i else 0 
                while i + l < n and s[i + l] == s[l]:
                    l += 1
                if i + l > r:
                    r = i + l 
                    c = i 
                z[i] = l 
            
            return z 
        
        z = z_arr(s)
        for i in range(k, n, k):
            if i + z[i] == n:
                return i // k 
        return (n + k - 1) // k
```

## 字符串哈希

### 1.字符串哈希 https://www.luogu.com.cn/problem/P3370

**思路：经典的字符串hash板子题。有两种方式：1.内置的set（对于过长的字符串来说，不是O(1)比对代价）；2.初始化base，建立映射，算每个字符串的hash值，最终进行个数统计**

**代码：**
```python

def II():

def LFI():
    return list(map(float, input().split()))

fmin = lambda x, y: x if x < y else y

mod = 10 ** 9 + 7

def val(c):
    if '0' <= c <= '9':
        return ord(c) - ord('0') + 1
    elif 'A' <= c <= 'Z':
        return ord(c) - ord('A') + 11
    else:
        return ord(c) - ord('a') + 37

def cal(s):
    base = 499 # 选质数
    ans = val(s[0])
    for i in range(1, len(s)):
        ans = (ans * base + val(s[i])) % mod # 取mod防爆

def solve():
    n = II()

        s = I()
        res.add(cal(s))
    print(len(res))

    solve()
```

### 2.找到字符串中第一个匹配项的下标 https://leetcode.cn/problems/find-the-index-of-the-first-occurrence-in-a-string/description/

**思路：字符串hash，当然这其实是kmp的板子题。字符串hash可以对s预处理出一个hash_arr数组，然后再配合pow_arr数组，就可以$O(1)$得到子串的hash值，然后再与t进行hash值匹配即可**

**代码：**
```python
class Solution:
    def strStr(self, s: str, t: str) -> int:
        n, m = len(s), len(t)
        
        base = 499
        mod = 10 ** 9 + 7

        pow_arr = [1] * (n + 1)
        for i in range(1, n + 1):
            pow_arr[i] = pow_arr[i - 1] * base % mod

        hash_arr = [0] * n
        hash_arr[0] = (ord(s[0]) - ord('a') + 1) % mod
        for i in range(1, n):
            hash_arr[i] = (hash_arr[i - 1] * base + ord(s[i]) - ord('a') + 1) % mod

        def hash_val(l, r):
            ans = hash_arr[r]
            if l > 0:
                ans = (ans - hash_arr[l - 1] * pow_arr[r - l + 1]) % mod
            return ans % mod

        ht = (ord(t[0]) - ord('a') + 1) % mod
        for i in range(1, m):
            ht = (ht * base + ord(t[i]) - ord('a') + 1) % mod

        i, j = 0, m - 1
        while j < n:
            if hash_val(i, j) == ht:
                return i
            i += 1
            j += 1

        return -1
```

### 3.重复叠加字符串匹配 https://leetcode.cn/problems/repeated-string-match/

**思路：思维 + 字符串匹配（KMP/字符串hash）。对于我们要叠加的次数来说，最多叠加$k = (a + b - 1)\ //\ 1$，也就是取$b / a$的上限次，再多叠加后面也是一样的，无法达成匹配。因此，我们只需要算上限叠加之后，当前匹配的长度在a * k的哪个区间中，就是答案了**

**代码：**
```python
class Solution:
    def repeatedStringMatch(self, s: str, t: str) -> int:
        a, b = len(s), len(t)
        k = (a + b - 1) // a
        s = s * (k + 1)
        
        n, m = len(s), len(t)
        base = 499
        mod = 10 ** 9 + 7

        pow_arr = [1] * (n + 1)
        for i in range(1, n + 1):
            pow_arr[i] = pow_arr[i - 1] * base % mod

        hash_arr = [0] * n
        hash_arr[0] = (ord(s[0]) - ord('a') + 1) % mod
        for i in range(1, n):
            hash_arr[i] = (hash_arr[i - 1] * base + ord(s[i]) - ord('a') + 1) % mod

        def hash_val(l, r):
            ans = hash_arr[r]
            if l > 0:
                ans = (ans - hash_arr[l - 1] * pow_arr[r - l + 1]) % mod
            return ans % mod

        ht = (ord(t[0]) - ord('a') + 1) % mod
        for i in range(1, m):
            ht = (ht * base + ord(t[i]) - ord('a') + 1) % mod

        i, j = 0, m - 1
        while j < n:
            if hash_val(i, j) == ht:
                return k if j < a * k else k + 1
            i += 1
            j += 1

        return -1
```
