```python
class Solution:
    def findOrder(self, numCourses: int, prerequisites: List[List[int]]) -> List[int]:
        g = [[] for _ in range(numCourses)]
        pre = [0] * numCourses
```

### 4.火星词典 https://leetcode.cn/problems/Jf1JuT/

**思路：难点在于怎么想到用拓扑排序的，代码还是比较好写。我们就两两比较相邻的字符串，如果发现是合法可比较的字符串。当两个字符出现不同时，就有左边字符字典序小于右边字符字典序，我们就可以连一条边，这样连完了之后，就构成了图。此时，我们发现这个图的拓扑排序，其实就是从小到大的字典序顺序，那就写个拓扑排序即可**

**代码：**
```python
class Solution:
    def alienOrder(self, words: List[str]) -> str:
        n = len(words)
        pre = [-1] * 26 
        for i in range(n):
            for c in words[i]:
                x = ord(c) - ord('a')
                pre[x] = 0 
        cnt = pre.count(0)
        
        g = [[] for _ in range(26)]
        for i in range(1, n):
            a, b = words[i - 1], words[i]
            if len(a) > len(b) and a[:len(b)] == b: # 这句非常重要，必加
                return ""
            for j in range(min(len(a), len(b))):
                if a[j] == b[j]:
                    continue 
                u, v = ord(a[j]) - ord('a'), ord(b[j]) - ord('a')
                if v not in g[u]:
                    g[u].append(v)
                    pre[v] += 1
                break
        
        q = deque([i for i, x in enumerate(pre) if x == 0])
        ans = []
        while q:
            for _ in range(len(q)):
                u = q.popleft()
                ans.append(chr(u + ord('a')))
                for v in g[u]:
                    pre[v] -= 1
                    if pre[v] == 0:
                        q.append(v)
        return ''.join(ans) if len(ans) == cnt else ""
```

### 5.戳印序列 https://leetcode.cn/problems/stamping-the-sequence/description/

**思路：和第4题很像，而且还更难。也是怎么想到拓扑排序的，然后建图该怎么建，并且要引入vis数组。具体流程如下：首先初始化入度数组，然后从[0, n - m]为开头，长为m的滑动窗口中进行判断，如果当前位置和印章位置元素相同，对应错误的入度-1，否则就建图（错误节点索引 -> 开头索引）；然后新建一个vis数组避免多次访问，以及拓扑排序经典的q双端队列，具体写bfs的拓扑排序逻辑时，添加上影响的i个索引位置以及vis数组访问即可；最终的答案就是拓扑序的逆序，如果有的话**

**代码：**
```python
class Solution:
    def movesToStamp(self, stamp: str, target: str) -> List[int]:
        m, n = len(stamp), len(target)
        pre = [m] * (n - m + 1)
        g = [[] for _ in range(n)]

        for i in range(n - m + 1):
            for j in range(m):
                if target[i + j] == stamp[j]: # 对应索引字符相同，正确
                    pre[i] -= 1
                else:
                    # 错误位置索引 -> 开头索引
                    g[i + j].append(i)
        
        vis = [False] * n # 同一个位置取消过，就不用再操作了
        q = deque([i for i, x in enumerate(pre) if x == 0])
        ans = []
        while q:
            for _ in range(len(q)):
                x = q.popleft() # 弹出开头索引（没有错误的）
                ans.append(x)
                for i in range(m):
                    if not vis[x + i]:
                        vis[x + i] = True 
                        for y in g[x + i]: # 去掉错误位置影响
                            pre[y] -= 1
                            if pre[y] == 0:
                                q.append(y)
        return ans[::-1] if len(ans) == n - m + 1 else []
```

### 6.最大食物链计数 https://www.luogu.com.cn/problem/P4017

### 7.喧闹和富有 https://leetcode.cn/problems/loud-and-rich/description/

**思路：有了上一题的基础，这题的拓扑排序就会好想一点，他是不断递推最大值的人，注意是人，而不是安静值。注意好这一点，后面就直接在拓扑排序板子里加逻辑即可**

**代码：**
```python
class Solution:
    def loudAndRich(self, richer: List[List[int]], quiet: List[int]) -> List[int]:
        n = len(quiet)
        pre = [0] * n
        ans = list(range(n))
        g = [[] for _ in range(n)]
        for x, y in richer:
            g[x].append(y)
            pre[y] += 1
        
        q = deque([i for i, x in enumerate(pre) if x == 0])
        while q:
            for _ in range(len(q)):
                x = q.popleft()
                for y in g[x]:
                    if quiet[ans[x]] < quiet[ans[y]]:
                        ans[y] = ans[x]
                    pre[y] -= 1
                    if pre[y] == 0:
                        q.append(y)
        return ans 
```

### 8.并行课程III https://leetcode.cn/problems/parallel-courses-iii/description/

**思路：和上面两个思路差不多，都是用拓扑排序进行信息的递推，最终根据题意输出ans**

**代码：**
```python
class Solution:
    def minimumTime(self, n: int, relations: List[List[int]], time: List[int]) -> int:
        pre = [0] * (n + 1)
        ans = [0] * (n + 1)
        g = [[] for _ in range(n + 1)]
        for x, y in relations:
            g[x].append(y)
            pre[y] += 1
        
        q = deque([i for i in range(1, n + 1) if pre[i] == 0])
        while q:
            for _ in range(len(q)):
                x = q.popleft()
                ans[x] += time[x - 1]
                for y in g[x]:
                    ans[y] = max(ans[x], ans[y])
                    pre[y] -= 1
                    if pre[y] == 0:
                        q.append(y)
        return max(ans)
```

### 9.参加会议的最多员工数 https://leetcode.cn/problems/maximum-employees-to-be-invited-to-a-meeting/

**思路：这题也是难在分析上面。首先，我们要让参加会议的员工最多，有两种方案：1.所有小环（长度为2）及其两端最长链之和；2.所有大环（长度>2）中，长度最长的那个，因为我们只能排一个大环，并且大环和小环不能排在一起**

**接下来就是如何找环以及如何确定环端点的最长链，应该用拓扑排序。首先用拓扑排序把所有环外面的点都给消去，并且消去的过程中不断递推环端点的最长链，然后把结果记录在mx数组中。最终遍历一遍pre数组，其中非零的，并且未被访问过的节点，算出其中的环长度以及环端点链长度之和。接着通过判断环长度来分别记录答案，最终输出两种方案中最大的那个即可**

**扩展：这题给的favorite数组可以看成是一个出度为1的特殊图，因此不需要再额外建图，正式学名叫基环树。为什么？因为他每个点出度为1，所以说必然会产生环**

**代码：**
```python
class Solution:
    def maximumInvitations(self, favorite: List[int]) -> int:
        n = len(favorite)
        pre = [0] * n 
        mx = [1] * n
        for i in range(n):
            pre[favorite[i]] += 1
        
        q = deque([i for i, x in enumerate(pre) if x == 0])
        while q:
            for _ in range(len(q)):
                u = q.popleft()
                v = favorite[u]
                mx[v] = max(mx[v], mx[u] + 1)
                pre[v] -= 1
                if pre[v] == 0:
                    q.append(v)
        
        ans = mmx = 0
        vis = [False] * n 
        for i in range(n):
            if pre[i] != 0 and not vis[i]:
                length = res = 0
                while not vis[i]:
                    length += 1
                    res += mx[i]
                    vis[i] = True
                    i = favorite[i]
                if length == 2:
                    ans += res 
                else:
                    mmx = max(mmx, length)
        return max(ans, mmx)
```

### 11.Endless Holidays https://atcoder.jp/contests/abc456/tasks/abc456_e

**思路：拓扑排序找环问题，不过加了一些限制条件。重点讲一下拓扑排序怎么找环的：不断删除外围，直到无法再删除，最终剩下节点如果入度不为0，说明有环，这个其实和第9题是一个思想**

```python
    g = [[] for _ in range(n)]
    for _ in range(m):
        u, v = MII()
        u -= 1; v -= 1
        g[u].append(v)
        g[v].append(u)

    w = II()
    s = [I().strip() for _ in range(n)]

    outdeg = [[0] * w for _ in range(n)]
    rg = [[[] for _ in range(w)] for _ in range(n)]

    for u in range(n):
        for d in range(w):
            nd = (d + 1) % w
            # 留在原地
            if s[u][nd] == 'o':
                outdeg[u][d] += 1
                rg[u][nd].append((u, d))
            # 去邻居
            for v in g[u]:
                if s[v][nd] == 'o':
                    outdeg[u][d] += 1
                    rg[v][nd].append((u, d))

    for u in range(n):
        for d in range(w):
            if outdeg[u][d] == 0:
                q.append((u, d))

    while q:
        u, d = q.popleft()
        removed[u][d] = True
        for pu, pd in rg[u][d]:
            if not removed[pu][pd]:
                outdeg[pu][pd] -= 1
                if outdeg[pu][pd] == 0:
                    q.append((pu, pd))

    for i in range(n):
        if s[i][0] == 'o' and not removed[i][0]:
```

## 最短路径问题

**Dijkstra算法：首先有两个前提：1.没有负边；2.源点到其他点的最短距离。然后比较经典的实现方法就是用邻接表 + 小根堆，直接写板子即可，不需要vis数组（小根堆天然有vis效果）**

**Floyd算法：首先有两个前提：1.没有负环；2.任意两点间的最短距离。然后比较经典的实现方法就是用邻接矩阵 + 枚举跳板k点，更新有跳板时的最短距离，也不需要用vis数组**

**A\*算法：首先有两个前提：1.没有负边；2.源点到终点的最短距离。这个算法是在Dijkstra算法上的扩展，主要是入堆的时候，在一般的dis上再加了期望函数，方便快速向终点查找，提升了速度。一般的，我们要根据题意选择合适的期望函数，满足期望函数距离 <= 真实距离。我们有三种期望函数可以选：1.曼哈顿距离；2.对角线距离；3.欧式距离**

**Bellman-Floyd + SPFA算法：就是在Bellman-Floyd算法的基础上加了双端队列标记的SPFA常数优化。首先有两个前提：1.可以有负边，不能有负环；2.可以判断负环是否存在（通过添加虚拟源点，然后用轮数判断）3.源点到其他点的最短距离。这个算法从源点出发，然后访问他相关的边，进行类似于dijkstra的距离更新，这样不断更新最多n - 1轮，因为我们最多访问n个点，最短路最长也就是n - 1条边，最终就可以得到结果**

### 1.邮递员送信 https://www.nowcoder.com/practice/2b0c636cf77d441fa96d40ac64290d39?channelPut=tracker2

**思路：两次dijkstra求和。题目强调了是单向边，并且是单源最短路径问题，所以说很容易能够想到用一次dijkstra，求得邮局到其他点的最短路径。但是题目还说了，每次送完信还要回到邮局，并且由于是单向边，所以说就不能再走第一次dijkstra的路，回到邮局就是我们要算其他位置到邮局的最短路径。**

**这其实是一个多到一的问题，我们只需要额外建一个反向图，这样问题就能转化成标准的一到多dijkstra问题，然后就和第一次dijkstra的代码是一样的。所以说，我们就可以把dijkstra写成函数封装起来，然后分别去处理原图和反图，最终相加得到答案即可**

**代码：**
```python
import sys
from collections import deque
from heapq import heappop, heappush

input = lambda: sys.stdin.readline().strip()

import math
inf = 10 ** 18

def I():
```

### 3.转换字符串的最小成本II https://leetcode.cn/problems/minimum-cost-to-convert-string-ii/description/?envType=daily-question&envId=2026-01-30

**思路：本题和他上一题差不多，就是从单个字符改成了字符串，所以说还是可以用两种方法来解决，一种是dijkstra，一种是floyd。不过最后算代价的变化就大了，因为不是定长的，我们不知道什么时候要进行替换，也就是说贪心是不行的，那就只能用dp来做。首先要预处理出可能的长度，然后写dp转移，最终查找代价，累加答案即可**

**代码：**
```python
class Solution:
    def minimumCost(self, source: str, target: str, original: List[str], changed: List[str], cost: List[int]) -> int:
        inf = 10 ** 18
        
        g = defaultdict(list)
        for u, v, w in zip(original, changed, cost):
            g[u].append((v, w))
        
        def dijkstra(u):
            dis = defaultdict(lambda: inf)
            dis[u] = 0 
            h = [(0, u)]
            while h:
                d, x = heappop(h)
                if d > dis[x]:
                    continue 
                for y, w in g[x]:
                    new_dis = d + w 
                    if new_dis < dis[y]:
                        dis[y] = new_dis 
                        heappush(h, (new_dis, y))
            return dis 

        dist = {}
        for x in original:
            dist[x] = dijkstra(x)
        
        # 预处理所有可能的长度，方便后续dp进行枚举
        words = set(original)
        lengths = set(len(w) for w in words)
        
        n = len(source)
        dp = [inf] * (n + 1) # dp[i]表示把source[0:i]变成target[0:i]的最小代价
        dp[0] = 0 # 初始化空串为0
        
        for i in range(n):
            if dp[i] == inf:
                continue
            
            # 如果相同则直接从上一个进行状态转移
            if source[i] == target[i]:
                dp[i + 1] = min(dp[i + 1], dp[i])
            
            # 如果不同就枚举可能的长度l，然后进行状态转移
            for l in lengths:
                if i + l > n:
                    continue
                u = source[i:i + l] 
                v = target[i:i + l] 

                if u in dist and v in dist[u]: 
                    dp[i + l] = min(dp[i + l], dp[i] + dist[u][v])
        
        return dp[n] if dp[n] < inf else -1
```

### 4.到达角落需要移除障碍物的最小数量 https://leetcode.cn/problems/minimum-obstacle-removal-to-reach-corner/description/

**思路：0-1bfs板子题，他这个还是在网格图里面的。具体算法流程和dijkstra比较相似，不过0-1bfs用的是双端队列，dijkstra用的是优先队列。具体流程如下：先建一个dis距离数组，然后把开始位置的$(i,\ j)$距离标记为0，之后新建双端队列，把$(d,\ i,\ j)$放进去，然后就开始写bfs流程，它里面的流程和dijkstra差不多，区别在于根据权重入队列这块：1.当$w == 0$时，从左边入队列；2.当$w == 1$时，从右边入队列。这个非常重要，它可以保证最终更新的答案是最小的，并且能够起到vis数组/hash表的记录访问**

**代码：**
```python
class Solution:
    def minimumObstacles(self, grid: List[List[int]]) -> int:
        inf = 10 ** 18
        m, n = len(grid), len(grid[0])

        dis = [[inf] * n for _ in range(m)]
        dis[0][0] = 0
        q = deque([(0, 0, 0)])
        while q:
            for _ in range(len(q)):
                d, i, j = q.popleft()
                if i == m - 1 and j == n - 1:
                    return d 
                if d > dis[i][j]:
                    continue 
                for x, y in (i + 1, j), (i - 1, j), (i, j + 1), (i, j - 1):
                    if 0 <= x < m and 0 <= y < n:
                        w = grid[x][y]
                        new_dis = d + w 
                        if new_dis < dis[x][y]:
                            dis[x][y] = new_dis 
                            if w == 0:
                                q.appendleft((new_dis, x, y))
                            else:
                                q.append((new_dis, x, y))
        return dis[-1][-1]
```

### 13.有限重边的最小阈值路径 https://leetcode.cn/problems/minimum-threshold-path-with-limited-heavy-edges/

**思路：二分答案 + 0-1bfs。首先可以观察出threshold具有单调性，threshold越大那么越能满足条件，反则反之。因此，题目转化为在只有0和1两个权值的情况下，算最短路并且结果要<= k的threshold，那就可以二分答案，然后check函数用0-1bfs来写，0-1bfs就是对dijkstra的优化，把优先队列改成双端队列，达到$O(n)$的效果，即去掉了logn维度**

**代码：**
```python
class Solution:
    def minimumThreshold(self, n: int, edges: List[List[int]], source: int, target: int, k: int) -> int:
        inf = 10 ** 18
        g = [[] for _ in range(n)]
        mx_w = 0 
        for x, y, w in edges:
            g[x].append((y, w))
            g[y].append((x, w))
            mx_w = max(mx_w, w)
        
        def check(mid):
            dis = [inf] * n
            dis[source] = 0 
            q = deque([(0, source)])

            while q:
                for _ in range(len(q)):
                    d, x = q.popleft()
                    if x == target:
                        return True 
                    for y, w in g[x]:
                        w = 1 if w > mid else 0 
                        new_dis = d + w 
                        if new_dis < dis[y]:
                            dis[y] = new_dis 
                            if w == 0:
                                q.appendleft((new_dis, y))
                            elif new_dis <= k: # 有限制条件
                                q.append((new_dis, y))
            return False

        l, r = 0, mx_w 
        while l <= r:
            mid = (l + r) >> 1
            if check(mid):
                r = mid - 1
            else:
                l = mid + 1
        return l if l <= mx_w else -1
```

### 14.购买苹果的最低成本II https://leetcode.cn/problems/minimum-cost-to-buy-apples-ii/description/

**思路：Dijkstra + 暴力枚举。简单说来就是建立两个图，表示从i到j没有苹果的图，以及从i到j有苹果的图，由于题目给定的是无向图，所以说i到j的最短路就是j到i的最短路，那我们在这两个图上面，每个i位置跑一个dijkstra，然后得到dis1以及dis2，接着不断取$min(p + dis1[i] + dis2[i])$，就可以得到当前位置的i答案了**

**代码：**
```python
inf = 10 ** 18 
def dijktra(g, start):
    dis = [inf] * len(g)
    dis[start] = 0 
    h = [(0, start)]

    while h:
        d, x = heappop(h)
        if d > dis[x]:
            continue 
        for y, w in g[x]:
            new_dis = d + w 
            if new_dis < dis[y]:
                dis[y] = new_dis 
                heappush(h, (new_dis, y))
    return dis 

class Solution:
    def minCost(self, n: int, prices: List[int], roads: List[List[int]]) -> List[int]:
        g1 = [[] for _ in range(n)]
        g2 = [[] for _ in range(n)]
        for x, y, cost, tax in roads:
            g1[x].append((y, cost))
            g1[y].append((x, cost))
            g2[x].append((y, cost * tax))
            g2[y].append((x, cost * tax))
        
        ans = [0] * n 
        for i, price in enumerate(prices):
            dis1 = dijktra(g1, i)
            dis2 = dijktra(g2, i)
            ans[i] = min(p + d1 + d2 for p, d1, d2 in zip(prices, dis1, dis2))
        return ans 
```

### 1.获取所有钥匙的最短路径 https://leetcode.cn/problems/shortest-path-to-get-all-keys/

**思路：分层图bfs，其实就是在一般的bfs上面额外添加了一个参数，多提升了一个维度。把原本的具体点到具体点，变成了从虚拟点到虚拟点，主要就是因为多出来的这层状态。因此，我们就额外需要一个vis的hash表来记录哪些状态已经访问过了**

**代码：**
```python
class Solution:
    def shortestPathAllKeys(self, grid: List[str]) -> int:
        m, n = len(grid), len(grid[0])
        si, sj = next((i, j) for i in range(m) for j in range(n) if grid[i][j] == '@')
        k = sum(v.islower() for row in grid for v in row)
        
        q = deque([(si, sj, 0)])
        vis = {(si, sj, 0)}
        ans = -1 
        while q:
            ans += 1
            for _ in range(len(q)):
                i, j, state = q.popleft()
                if state == (1 << k) - 1:
                    return ans 
                for x, y in (i + 1, j), (i - 1, j), (i, j + 1), (i, j - 1):
                    if 0 <= x < m and 0 <= y < n and grid[x][y] != '#':
                        c, nxt = grid[x][y], state
                        if c.isupper() and state & (1 << (ord(c) - ord('A'))) == 0:
                            continue 
                        if c.islower():
                            nxt |= 1 << (ord(c) - ord('a'))
                        if (x, y, nxt) not in vis:
                            vis.add((x, y, nxt))
                            q.append((x, y, nxt))
        return -1
```

### 2.电动车游城市 

**思路：分层图dijkstra，就是在一般的dijkstra算法基础上，新增一个参数，扩展一个维度，然后就是注意分层图dijkstra新增的参数要逐个增加状态，慢慢把所有可能性都放到算法里面跑，最终会自动找到答案。还有就是，dijkstra算法不需要vis的hash表，因为优先队列，即堆具备vis的功能**

**代码：**
```python
class Solution:
    def electricCarPlan(self, paths: List[List[int]], cnt: int, start: int, end: int, charge: List[int]) -> int:
        n = len(charge)
        g = [[] for _ in range(n)]
        for u, v, w in paths:
            g[u].append((v, w))
            g[v].append((u, w))

        dis = [[inf] * (cnt + 1) for _ in range(n)] # 多一个状态
        dis[start][0] = 0
        h = [(0, start, 0)]

        while h:
            d, x, e = heappop(h)
            if x == end:
                return d
            if e < cnt: # 分层图dijkstra必要操作，每次状态+1，后续让算法自动找最优解
                new_dis = d + charge[x]
                if new_dis < dis[x][e + 1]:
                    dis[x][e + 1] = new_dis
                    heappush(h, (new_dis, x, e + 1))
            for y, w in g[x]:
                if y == x: # 有自环需要判断一下，这里其实可以不要，对于回溯，dfs这种才必须要写y != fa
                    continue
                new_dis = d + w
                if e >= w and new_dis < dis[y][e - w]:
                    dis[y][e - w] = new_dis
                    heappush(h, (new_dis, y, e - w))
        return -1
```

### 4.打砖块 https://leetcode.cn/problems/bricks-falling-when-hit/description/

**思路：可以用并查集写，但是会很复杂。这里采用左神的思路：首先把所有炮弹位置-1；然后dfs标记天花板位置，之后再进行时光倒流，逆向+1；然后判断当前位置是砖块的情况下，自己是不是天花板或者邻居有天花板相连的2；然后就可以进行dfs算大小，最终当前位置的答案就是其dfs的大小 - 1，也就是减去自身的1大小。最终输出结果数组ans即可**

**代码：**
```python
class Solution:
    def hitBricks(self, grid: List[List[int]], hits: List[List[int]]) -> List[int]:
        m, n = len(grid), len(grid[0])

        for x, y in hits:
            grid[x][y] -= 1
        
        def dfs(i, j):
            size = 1
            grid[i][j] = 2 
            for x, y in (i + 1, j), (i - 1, j), (i, j + 1), (i, j - 1):
                if 0 <= x < m and 0 <= y < n and grid[x][y] == 1:
                    size += dfs(x, y)
            return size
        
        for j in range(n):
            if grid[0][j] == 1:
                 dfs(0, j)
        
        l = len(hits)
        ans = [0] * l
        for k in range(l - 1, -1, -1):
            i, j = hits[k]
            grid[i][j] += 1
            if grid[i][j] != 1: # 这行很重要，如果当前不是砖块，就不用操作了
                continue
            if i == 0:
                ans[k] = dfs(i, j) - 1
            else:
                for x, y in (i + 1, j), (i - 1, j), (i, j + 1), (i, j - 1):
                    if 0 <= x < m and 0 <= y < n and grid[x][y] == 2:
                        ans[k] = dfs(i, j) - 1
                        break 
        return ans
```

### 6.地图分析 https://leetcode.cn/problems/as-far-from-land-as-possible/description/

### 7.二维网格中探测环 https://leetcode.cn/problems/detect-cycles-in-2d-grid/?envType=daily-question&envId=2026-04-26

**思路：dfs + vis标记。找环问题不能用bfs，这个无法判断是否成环了，本题算一个很经典的板子题，可以记一下**

**代码：**
```python
class Solution:
    def containsCycle(self, grid: List[List[str]]) -> bool:
        m, n = len(grid), len(grid[0])
        
        vis = [[False] * n for _ in range(m)]
        def dfs(i: int, j: int, fai: int, faj: int) -> bool:
            if vis[i][j]:
                return True
            vis[i][j] = True
            flag = False
            for x, y in (i + 1, j), (i - 1, j), (i, j + 1), (i, j - 1):
                if x == fai and y == faj:
                    continue
                if 0 <= x < m and 0 <= y < n and grid[x][y] == grid[i][j]:
                    flag |= dfs(x, y, i, j)
            return flag

        for i in range(m):
            for j in range(n):
                if not vis[i][j] and dfs(i, j, -1, -1):
                    return True
        return False
```

### 1.【模板】最小生成树 https://www.luogu.com.cn/problem/P3366

**思路：一般是用kruskal算法，也就是边权法操作，用并查集比较好写。具体来说：首先不需要建图，直接收集所有边及边权，然后按照边权从小到大进行排序；排序好了之后，用并查集来合并可以收集的边，如果合并的双方在一个集合中，那么加了边之后必然会导致环的出现，应该跳过，否则就可以加边，然后进行合并；优化的话，就可以用一个cnt变量记录加入了多少条边了，我们只需要n - 1条边即可**

**prim算法就需要建图（边权在前，端点在后，便于后续小根堆操作），然后用一个点集合，一个边权小根堆，初始化把1节点加进来，然后把他连接的其他边给加到小根堆。然后就不断从小根堆中弹出小边权，判断端点是否出现在点集合中，如果存在就跳过，否则加入到点集合中，更新ans，把新加入这个点有关的边加入小根堆中，这样不断操作下去，直到节点够了或者边权小根堆空了就结束**

## 广搜bfs

### 1.单词接龙（双向广搜） https://leetcode.cn/problems/word-ladder/

**思路：首先特判单词表中有没有结果单词，如果没有就直接返回0即可。然后我们就可以把结果单词以外的其他单词进行处理，具体来说就是每个位置枚举另外的25个字母，然后再进行拼接，如果单词表中有这个单词，那原单词就可以到达这个单词，我们就在其之间连一条线**

**预处理完所有单词并建好图之后，我们就可以用双向广搜了，也就是开始与结束点同时进行搜索，具体流程如下：首先建两个双向队列，然后建两个hash表，用于储存当前单词深度是多少；之后就进行双向广搜，每次处理队列中元素少的一方，为了简化书写，我们默认q1的元素更少，如果q1更多的话，我们就提前交换一下两者的队列以及hash表即可，能够大大简化逻辑**

**之后就正常写bfs，如果发现当前处理的元素出现在了另一个集合中，说明出现了交集，直接返回答案即可；否则，就更新当前单词的距离，然后放进q1队列，继续跑双向bfs，直到返回答案**

**代码：**
```python
class Solution:
    def ladderLength(self, beginWord: str, endWord: str, wordList: List[str]) -> int:
        if endWord not in wordList:
            return 0
        
        all_words = set(wordList + [beginWord])
        neighbors = defaultdict(list)
        
        for word in all_words:
            for i in range(len(word)):
                for c in 'abcdefghijklmnopqrstuvwxyz':
                    if c == word[i]:
                        continue
                    nxt = word[:i] + c + word[i + 1:]
                    if nxt in all_words:
                        neighbors[word].append(nxt)
        
        q1, q2 = deque([beginWord]), deque([endWord])
        dist1, dist2 = {beginWord: 1}, {endWord: 1}
        
        while q1 and q2:
            if len(q1) > len(q2):
                q1, q2 = q2, q1
                dist1, dist2 = dist2, dist1
            
            for _ in range(len(q1)):
                word = q1.popleft()
                for nxt in neighbors[word]:
                    if nxt in dist1:
                        continue

                    if nxt in dist2:
                        return dist1[word] + dist2[nxt]

                    dist1[nxt] = dist1[word] + 1
                    q1.append(nxt)
        return 0
```

### 2.E - Multiple-Free Sequences（状态广搜） https://atcoder.jp/contests/abc446/tasks/abc446_e

**思路：这题难在分析，首先我们要把$s_n = As_{n - 1} + Bs_{n - 2}$因子中不包含m，转化成$s_n = (As_{n - 1} + Bs_{n - 2})\ \%\ m$不为0。然后注意到状态转移有$(i,\ j) \ ->\ (j,\ k)$，这里的$z = Ai + Bj$，怎么注意到的？因为题目天然有递推关系，从前两个就可以逐步向后推了**

**那此时我们应该建图广搜，然后再判断$m^2$个状态中，有没有到达0的可能性吗？不行，这样复杂度太高了，反向思考一下：从基础的有0状态，倒推这些危险状态能到达哪些状态，最后再进行收集，这种方法的复杂度明显更好，因为我们就建反图来操作。然后要用到一个技巧，那就是维度压缩。对于$(i, j)$来说，我们可以压缩成$v = i * m + j$，这个和二维数组里面的压缩是一个方法。最后广搜一遍，统计答案即可**

**代码：**
```python
import sys
from collections import deque

import math
inf = 10 ** 18

def I():
    return input()

def II():
    return int(input())

def MII():
    return map(int, input().split())

def LI():
    return input().split()

def LII():
    return list(map(int, input().split()))

def LFI():
    return list(map(float, input().split()))

fmax = lambda x, y: x if x > y else y
fmin = lambda x, y: x if x < y else y
isqrt = lambda x: int(math.sqrt(x))

'''

'''

def solve():
    m, a, b = LII()

    g = [[] for _ in range(m * m)]
    for i in range(m):
        for j in range(m):
            k = (a * j + b * i) % m
            g[j * m + k].append(i * m + j) # 维度压缩，建反图

    seen = [False] * (m * m) # 也是维度压缩
    q = deque()

    # 标记出有0的危险状态
    for i in range(m):
        for j in range(m):
            if i == 0 or j == 0:
                u = i * m + j # 维度压缩
                seen[u] = True
                q.append(u) # 危险状态入队

    # 广搜，把所有和危险状态的有关的状态都标记出来
    while q:
        for _ in range(len(q)):
            u = q.popleft()
            for v in g[u]:
                if not seen[v]:
                    seen[v] = True
                    q.append(v)

    ans = 0
    for i in range(m):
        for j in range(m):
            if not seen[i * m + j]:
                ans += 1

    print(ans)

t = 1
# t = II()
for _ in range(t):
    solve()
```

```python
        n = len(nums)
        groups = defaultdict(list)
        for i, x in enumerate(nums):
            for p in prime_factors[x]:
                groups[p].append(i)
        
        ans = 0
        vis = [False] * n
        vis[0] = True
        q = deque([0])

        while q:
            for _ in range(len(q)):
                i = q.popleft()
                if i == n - 1:
                    return ans
                idx = groups[nums[i]]
                idx.append(i + 1)
                if i:
                    idx.append(i - 1)
                for j in idx:
                    if not vis[j]:
                        vis[j] = True
                        q.append(j)
                idx.clear()
            ans += 1
```
