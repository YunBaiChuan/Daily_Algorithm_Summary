## 回溯问题

**通用思路：回溯三板斧：构建现场，递归，恢复现场**

### 1.游游的二进制树 https://www.nowcoder.com/practice/832141cc566a49ad923bb9ad5bb80fac?channelPut=tracker2

**思路：题目说了n的范围较小，所以说可以用回溯来找到所有可能的路径并进行统计。首先我们可以做一个剪枝，如果当前的值已经>r了，就直接结束。然后还需要注意题目说了路径长度不能为1，因此要用一个depth变量来记录下长度**

**接下来就可以正常写回溯了，不过在统计答案的时候需要注意下，当前值处于l和r之间就可以进行答案统计，但是统计完了之后不要直接返回，因为有可能l和r的范围比较大，能够有多个满足条件的答案，这个和一般的回溯输出路径有所不同。**

**那写回溯就是三件套了：构建现场，递归，恢复现场。这里我用的是位运算来写的，这样会更快一些。然后再以每个节点作为开头，进行一次dfs，最终输出结果即可**

**代码：**
```python
import sys
input = lambda: sys.stdin.readline().strip()

import math
inf = 10 ** 18

def I():
    return input()

def II():
    return int(input())

def MII():
    return map(int, input().split())

def GMI():
    return map(lambda x: int(x) - 1, input().split())

def LI():
    return input().split()

def LII():
    return list(map(int, input().split()))

def LFI():
    return list(map(float, input().split()))

fmax = lambda x, y: x if x > y else y
fmin = lambda x, y: x if x < y else y
isqrt = lambda x: int(math.sqrt(x))

'''

'''

def solve():
    n, l, r = LII()
    a = [0] + list(map(int, I()))
    g = [[] for _ in range(n + 1)]
    for _ in range(n - 1):
        u, v = MII()
        g[u].append(v)
        g[v].append(u)

    def dfs(u, fa):
        nonlocal tmp, ans, depth
        if tmp > r:
            return

        if l <= tmp <= r and depth > 1:
            ans += 1

        for v in g[u]:
            if v == fa:
                continue
            depth += 1
            tmp = tmp << 1 | a[v]
            dfs(v, u)
            tmp >>= 1
            depth -= 1

    ans = 0
    for i in range(1, n + 1):
        tmp = a[i]
        depth = 1
        dfs(i, -1)

    print(ans)

t = 1
# t = II()
for _ in range(t):
    solve()
```

### 2.N皇后问题II https://leetcode.cn/problems/n-queens-ii/

**思路：经典回溯题目，有两种实现方法：1.定长数组；2.位运算。不过要注意：用定长数组实现的其实是回溯，而用位运算实现的不是回溯，为什么？对于数组来说，无法做到像位运算那样的直接$O(1)$计算出还有哪些位置能放，只能遍历判断，然后递归，再回溯。但是位运算每次可以算出能放的位置，因此不需要回溯，不过整体思路都是递归来做**

**代码：定长数组实现**
```python
class Solution:
    def totalNQueens(self, n: int) -> int:
        if n < 1: # 本题可不用此特判
            return 0
        path = [-1] * n # 回溯数组，后面可以覆盖对应值来替代恢复现场
        return self.f1(0, path, n)
    
    # 枚举从i到n个皇后的列摆放位置，行不用管，已经是一行一个了
    def f1(self, i: int, path: list[int], n: int) -> int:
        if i == n:
            return 1 
        ans = 0 
        for j in range(n):
            if self.check(path, i, j):
                path[i] = j # 由于是定长数组，所以说同时具有构建现场和恢复现场的作用
                ans += self.f1(i + 1, path, n) # 注意这里是n，末尾应该是问题规模
        return ans 
    
    # 判断在前面path[0...i - 1]放的皇后前提下，当前位置(i, j)能不能放皇后
    def check(self, path: list[int], i: int, j: int) -> bool:
        for k in range(i):
            if j == path[k] or abs(i - k) == abs(j - path[k]): # 列及45°对角线判断
                return False 
        return True 
```

**代码：位运算实现**
```python
class Solution:
    def totalNQueens(self, n: int) -> int:
        if n < 1:
            return 0 
        limit = (1 << n) - 1
        return self.f2(limit, 0, 0, 0)
    
    def f2(self, limit, col, left, right):
        if col == limit:
            return 1 

        ban = col | left | right # 列和对角线或起来就是不能放的1位置
        candidate = limit & (~ban) # 由于后面要算lowbit，因此这里对ban取个反再交

        ans = 0 
        while candidate:
            place = candidate & (-candidate) # 取lowbit，表示可以放的列
            candidate ^= place # 当前递归层放place这个列
            ans += self.f2(limit, (col | place), (left | place) >> 1, (right | place) << 1)
        
        return ans 
```

### 4.全排列I https://leetcode.cn/problems/permutations/description/

**思路：可以用交换法，也可以用定长数组，但如果用后者的话，要额外创建一个used数组或者hash表，记录值是否使用过，所以说建议写交换法，好写且不浪费空间**

**代码：**
```python
class Solution:
    def permute(self, nums: List[int]) -> List[List[int]]:
        n = len(nums)
        ans = []

        def dfs(i):
            if i == n:
                ans.append(nums.copy())
                return 
            for j in range(i, n):
                nums[i], nums[j] = nums[j], nums[i]
                dfs(i + 1)
                nums[i], nums[j] = nums[j], nums[i]
        dfs(0)
        return ans 
```

### 5.全排列II https://leetcode.cn/problems/permutations-ii/description/

**思路：这个是基于前面交换法，加了一个st的hash表，这个很巧妙，他就是每个递归层只用一遍那个数，这样就不会造成重复，而且用set去重的话，就不需要事先排好序了，强烈推荐写**

**代码：**
```python
class Solution:
    def permuteUnique(self, nums: List[int]) -> List[List[int]]:
        n = len(nums)
        ans = []

        def dfs(i):
            if i == n:
                ans.append(nums.copy())
                return 
            st = set()
            for j in range(i, n):
                if nums[j] not in st:
                    st.add(nums[j])
                    nums[i], nums[j] = nums[j], nums[i]
                    dfs(i + 1)
                    nums[i], nums[j] = nums[j], nums[i]
        dfs(0)
        return ans 
```

### 6.单词搜索 https://leetcode.cn/problems/word-search/description/

**思路：每个位置回溯递归搜索一下，如果有一个位置满足条件就返回True**

**代码：**
```python
class Solution:
    def exist(self, board: List[List[str]], word: str) -> bool:
        m, n = len(board), len(board[0])
        def dfs(i, j, k):
            if board[i][j] != word[k]:
                return False 
            if k == len(word) - 1:
                return True 
            board[i][j] = ''
            for x, y in (i + 1, j), (i - 1, j), (i, j + 1), (i, j - 1):
                if 0 <= x < m and 0 <= y < n and dfs(x, y, k + 1): # 这里要把dfs放在if上面才行，为什么？因为我们要搜素所有方向的可能性，如果放在return里面，遇到的第一条路径就是False，那么他就会直接返回，不会去搜索其他剩余的、可能满足条件的路径了
                    return True 
            board[i][j] = word[k]
            return False 
        return any(dfs(i, j, 0) for i in range(m) for j in range(n))
```

### 7.长度为 n 的开心字符串中字典序第 k 小的字符串 https://leetcode.cn/problems/the-k-th-lexicographical-string-of-all-happy-strings-of-length-n/description/?envType=daily-question&envId=2026-03-14

**思路：暴力回溯，固定长度类型，不用显式写恢复现场。并且从小到大填，最终字典序必然是从小到大排好序的**

**代码：**
```python
class Solution:
    def getHappyString(self, n: int, k: int) -> str:
        a = ['a', 'b', 'c']
        res = []
        path = [''] * n
        def dfs(i):
            if i == n:
                if all(x != y for x, y in pairwise(path)):
                    res.append(''.join(path))
                return 
            for j in range(3):
                path[i] = a[j]
                dfs(i + 1)
        dfs(0)
        return res[k - 1] if len(res) >= k else ""
```

## 双向广搜

**本质：双向广搜是对全局广搜的优化，其最终效果与全局广搜一致。并且注意，他的最终状态不是相加，而是$2^{n / 2} * 2^{n / 2}$，即$2^n$，状态数和全局广搜一致。但他在找答案/合并阶段用了排序双指针优化，这就比直接全局广搜，然后一个一个检查更好，不管是时间复杂度还是空间复杂度，都是更优的**

### 1.世界冰球锦标赛 https://www.luogu.com.cn/problem/P4799

### 2.牛牛的背包问题 https://www.nowcoder.com/practice/bf877f837467488692be703735db84e6?tpId=0&difficulty=&judgeStatus=&tags=&title=%E7%89%9B%E7%89%9B%E7%9A%84%E8%83%8C%E5%8C%85&sourceUrl=&gioEnter=menu

**思路：这两题的代码是一样的，都是用的双向广搜。具体来说就是：首先观察题目数据范围，发现用不了背包dp，会爆内存，而且时间复杂度也不行；然后考虑暴力搜索，因为题目的n相比之下会更小，全局广搜的时间复杂度为$O(2^n)$，此时的n为40，如果那么量级就来到了恐怖的$1 * 10^{12}$，必然会超时；那就需要用到双向广搜了，我们分成三个部分：1.左边一半广搜；2.右边一半广搜；3.左右广搜后的结果分别排序，最终用双指针进行整合。那此时这三部分的时间复杂度之和，也就是双向广搜的最终时间复杂度，也就是$O(3 * 2^{n / 2})$，这个时间复杂度已经非常好了**

**代码：**
```python
import sys
input = lambda: sys.stdin.readline().strip()

import math
inf = 10 ** 18

def I():
    return input()

def II():
    return int(input())

def MII():
    return map(int, input().split())

def LI():
    return input().split()

def LII():
    return list(map(int, input().split()))

def LFI():
    return list(map(float, input().split()))

fmax = lambda x, y: x if x > y else y
fmin = lambda x, y: x if x < y else y
isqrt = lambda x: int(math.sqrt(x))

'''

'''

def solve():
    n, w = MII()
    a = LII()

    def func(i, e, s, nums):
        if s > w:
            return
        if i == e:
            nums.append(s)
        else:
            func(i + 1, e, s, nums)
            func(i + 1, e, s + a[i], nums)

    lnums, rnums = [], []
    func(0, n // 2, 0, lnums)
    func(n // 2, n, 0, rnums)

    lnums.sort()
    rnums.sort()

    ans = 0
    p, q = len(lnums), len(rnums)
    i, j = 0, q - 1
    while i < p and j >= 0:
        while j >= 0 and lnums[i] + rnums[j] > w:
            j -= 1
        ans += (j + 1)
        i += 1
    print(ans)

t = 1
# t = II()
for _ in range(t):
    solve()
```

### 3.最接近目标值的子序列和 https://leetcode.cn/problems/closest-subsequence-sum/description/

**思路：这题也是n比较小，但是w比较大，所以说还是用双向广搜。前面的板子一直，主要是最后收集答案的排序双指针那里，需要修改一下，以匹配题目的要求**

**代码：**
```python
class Solution:
    def minAbsDifference(self, nums: List[int], goal: int) -> int:
        inf = 10 ** 18 
        n = len(nums)
        
        def func(i, e, s, a):
            if i == e:
                a.append(s)
            else:
                func(i + 1, e, s, a)
                func(i + 1, e, s + nums[i], a)
        
        lnums, rnums = [], []
        func(0, n // 2, 0, lnums)
        func(n // 2, n, 0, rnums)

        lnums.sort()
        rnums.sort()

        p, q = len(lnums), len(rnums)
        i, j = 0, q - 1
        ans = inf
        
        while i < p and j >= 0:
            cur = lnums[i] + rnums[j]
            ans = min(ans, abs(cur - goal))

            if cur == goal:
                return 0
            elif cur < goal:
                i += 1  
            else:
                j -= 1    
                
        return ans
```

## 树的dfs

### 1.从根到叶的二进制数之和 https://leetcode.cn/problems/sum-of-root-to-leaf-binary-numbers/description/?envType=daily-question&envId=2026-02-24

**思路：从根到叶子做一个dfs，然后nonlocal的ans来收集到达叶子节点后的结果即可，这里更好的写法是用位运算，可以好好学习一下**

**代码：**
```python
class Solution:
    def sumRootToLeaf(self, root: Optional[TreeNode]) -> int:
        ans = 0
        def dfs(node, num):
            nonlocal ans
            if not node:
                return 
            num = num << 1 | node.val
            if not node.left and not node.right:
                ans += num
                return 
            dfs(node.left, num)
            dfs(node.right, num)
        dfs(root, 0)
        return ans
```
