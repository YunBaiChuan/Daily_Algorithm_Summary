**思路：外面两层for的暴力枚举字串，这个没法优化，需要优化的是内部的for算最小代价，这个比较巧妙。具体来说就是：从长度进行优化，令$c0 = 0$; $c1 = 0$表示两种不同的交替形式，然后就可以根据长度$j - i$的奇偶性加上异化，来判断当前位置匹配格式所需要花费的代价，ans每次累加更小的字串代价，就可以得到最终ans了**

**代码：**
```python
import sys
input = lambda: sys.stdin.readline().strip()

import math
inf = 10 ** 18

def I():
    return input()

def II():
    return int(input())

def MII():
    return map(int, input().split())

def LI():
    return input().split()

def LII():
    return list(map(int, input().split()))

def LFI():
    return list(map(float, input().split()))

fmax = lambda x, y: x if x > y else y
fmin = lambda x, y: x if x < y else y
isqrt = lambda x: int(math.sqrt(x))

'''

'''

def solve():
    s = I()
    n = len(s)
    a = list(map(int, s))

    ans = 0
    for i in range(n):
        c0 = c1 = 0
        for j in range(i, n):
            if (j - i) & 1:
                c0 += a[j] ^ 1
                c1 += a[j] ^ 0
            else:
                c0 += a[j] ^ 0
                c1 += a[j] ^ 1
            ans += fmin(c0, c1)
    print(ans)

t = 1
# t = II()
for _ in range(t):
    solve()
```
