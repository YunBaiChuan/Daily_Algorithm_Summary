## 第一题：距离最小相等元素查询 https://leetcode.cn/problems/closest-equal-element-queries/description/?envType=daily-question&envId=2026-04-16

**思路：和昨天那题最短距离差不多，不过是增强版，需要用到hash表和二分查找。怎么想到二分查找？比较简单，就是每个固定的查询必然在hash表中出现，并且我们只需要找到前一个和后一个数，算这两个就行，其他位置必然会比这两个位置更差**

**代码：**
```python
class Solution:
    def solveQueries(self, nums: List[int], queries: List[int]) -> List[int]:
        m = len(nums)
        d = defaultdict(list)
        for i, x in enumerate(nums):
            d[x].append(i)

        n = len(queries)
        ans = [-1] * n 
        for i, x in enumerate(queries):
            num = nums[x]
            row = d[num]

            rn = len(row)

            if len(row) < 2:
                continue

            p = bisect_left(row, x)
            l, r = (p - 1 + rn) % rn, (p + 1) % rn
            pl, pr = row[l], row[r]
            if pl > x:
                res1 = min(pl - x, m - (pl - x))
            else:
                res1 = min(x - pl, m - (x - pl))
            if pr > x:
                res2 = min(pr - x, m - (pr - x))
            else:
                res2 = min(x - pr, m - (x - pr))

            ans[i] = min(res1, res2)
            
        return ans 
```

## 第二题：小红树上染色 https://www.nowcoder.com/practice/e89ce3d055024e3c996b629f8286c1fc?channelPut=tracker2

**思路：树上dp。方法容易看出来，不过状态转移需要好好分析一下。定义$dp[i][j]$表示前$i$个节点中，染色为$j = 0, 1$，即白色和红色的方案数。最开始方案数都是0，然后有初始化为$dp[u][0] = dp[u][1] = 1$，接着最重要的状态转移方程就是：1.u为白色，那么v必然要染成红色，即$dp[u][0] = dp[u][0] * dp[v][1] % mod$；2.u为红色，那么v可以是白色、红色，即$dp[u][1] = dp[u][1] * (dp[v][0] + dp[v][1]) % mod$**

**最终的答案就是根节点的两个方案数之和，注意是根节点，然后由于没有返回值，所以说不需要写@lru_cache，这题算是很经典的树上dp方案数问题了**
