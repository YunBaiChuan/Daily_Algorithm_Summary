## 第一题：香槟塔 https://leetcode.cn/problems/champagne-tower/description/?envType=daily-question&envId=2026-02-14

**思路：严格来说，应该是一道dp。这里用了滚动数组nxt来实现，和一般固定的m * n动态数组不同，这里每层的n是不一样的。接着，他的状态转移方程其实就是，当上一层如果有多余的酒流下来即$x > 1$，那么就可以转移到j和j + 1上面，之后滚动更新即可**

**这里注意下，由于题目只要求算到query_row，我们又是初始化拿poured算的，那就可以只算到query_row**

**代码：**
```python
class Solution:
    def champagneTower(self, poured: int, query_row: int, query_glass: int) -> float:
        cur = [float(poured)]
        for i in range(1, query_row + 1):
            nxt = [0.0] * (i + 1)
            for j, x in enumerate(cur):
                if x > 1:
                    nxt[j] += (x - 1) / 2
                    nxt[j + 1] += (x - 1) / 2 
            cur = nxt 
        return min(1.0, cur[query_glass])
```

## 第二题：国王游戏 https://www.nowcoder.com/practice/145c3426577f40f8b2b5e51ca6f61523?channelPut=tracker2

**思路：贪心。题目强调了最小化最大值这个要求，不知道有没有同学用二分答案呢？但你仔细分析一下可以发现，这题不具备单调性，比如钱越多，越怎么样？没有单调性是用不了二分的。考虑暴力，直接全排列n个大臣，然后再一个一个算，那么时间复杂度就来到了惊人的$O(n * n!)$，那肯定过不了。**

**所以，正确的解法是贪心。首先第一直觉来说：$a_i$越小，$b_i$越大，我们倾向于放在后面，那感觉贪心和排序有关。然后我就开始猜了，试了下用直接、$a_i - b_i$、$b_i - a_i$、$a_i\ /\ b_i$、$b_i\ /\ a_i$、$a_i * b_i$来进行排序贪心，最后发现$a_i * b_i$贪心是对的。接着用前缀乘积算一遍结果，其中的最大值就是答案**

**代码：**
