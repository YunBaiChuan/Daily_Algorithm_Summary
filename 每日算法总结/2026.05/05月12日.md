## 第一题：完成所有任务的最少初始能量 https://leetcode.cn/problems/minimum-initial-energy-to-finish-tasks/description/?envType=daily-question&envId=2026-05-12

**思路：排序贪心 + 二分答案/公式推导。首先观察题目，可以发现这个能量具有单调性，初始能量越多，那么完成的任务越多；反之越少；然后就是check函数怎么写了，我们希望task[0]尽可能小，而task[1]尽可能大，所以说两者相减得到整体越小越好，因此就按照这个进行排序，最后跑一个二分答案即可**

**代码：**
```python
class Solution:
    def minimumEffort(self, tasks: List[List[int]]) -> int:
        n = len(tasks)
        tasks.sort(key=lambda x: x[0] - x[1])

        def check(mid):
            for x, y in tasks:
                if mid < y:
                    return False 
                mid -= x 
            return True

        l, r = 1, 10 ** 18 
        while l <= r:
            mid = (l + r) >> 1
            if check(mid):
                r = mid - 1
            else:
                l = mid + 1
        return l 
```

## 第二题：Bring It To Back https://codeforces.com/gym/106164/problem/B

**思路：思维。大致有三种情况：1.特判n == 2时，结果只和m的奇偶性有关；2.m < n时，结果就是[n...n - m + 1 1 2.. n - m]；3.m >= n时，结果必然是[n n - 1 n - 2...1]，因为我们首先可以用n - 1次操作将他变成[1 2 3...n]的合法要求，之后再不断交换n - 1和n这两个数，就可以保持合法要求（这是n - 1和m奇偶相同的情况；否则可以先用3次操作把末尾[3, 2, 1]变成[1, 2, 3]，使得奇偶一致）**
